# UniMeet (유니밋) 개발용 PRD v2

> 선문대학교 내·외국인 학생 통합 동아리 플랫폼  
> 기준 산출물: 개발용 PRD  
> 기술 기준: React Native + Expo + Firebase Serverless  
> 작성 기준: 기존 PRD, Claude 프로토타입 문서, Stitch 디자인 export 참고

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | UniMeet (유니밋) |
| 플랫폼 | 모바일 앱 (iOS / Android) |
| 대상 학교 | 선문대학교 |
| 주요 사용자 | 내국인 학생, 외국인 유학생, 동아리 회장, 총동아리회 운영자, 학교 관리자 |
| MVP 기술 스택 | Expo, React Native, Firebase Auth, Firestore, Firebase Storage 최소 사용, Google Translate API |
| MVP 핵심 목표 | 동아리 탐색, 지원서 제출, 회장 지원서 검토, 운영자 동아리/회장 권한 관리, 앱 내부 알림 |

UniMeet은 선문대학교 학생들이 동아리를 쉽게 찾고 지원하며, 동아리 회장과 총동아리회 운영자가 모집 과정을 디지털로 관리할 수 있도록 돕는 모바일 앱이다.

기존 Claude 기반 프로토타입은 화면 분위기와 일부 기능 아이디어 참고 자료로만 사용한다. 실제 MVP 화면 구조와 기능 범위는 본 PRD v2를 기준으로 재정의한다.

## 2. 문제 정의

### 2-1. 사용자 문제

- 동아리 모집 정보가 카카오톡, 인스타그램, 오프라인 홍보물 등에 분산되어 있다.
- 유학생과 신입생은 동아리 정보를 찾기 어렵고, 한국어 중심 공고 때문에 접근성이 낮다.
- 지원 절차가 오프라인, 구글폼, DM 등으로 흩어져 지원 상태를 확인하기 어렵다.
- 회장은 지원서 확인, 합격/불합격 안내, 홍보글 관리를 여러 도구로 처리해야 한다.
- 총동아리회 운영자는 동아리 생성과 회장 권한 관리를 수기로 처리하는 부담이 크다.

### 2-2. 서비스 목표

- 학생이 동아리를 탐색하고 지원서를 제출하는 과정을 앱 안에서 완료한다.
- 회장이 홍보카드와 지원서를 한 곳에서 관리한다.
- 운영자가 동아리를 생성하고 회장 권한을 부여하는 관리 흐름을 디지털화한다.
- 공지와 동아리 소개에 한해 자동 번역을 제공하여 유학생 접근성을 개선한다.
- MVP에서는 채팅, 푸시 알림, 행정 서류, 통계 기능을 제외하고 핵심 모집 흐름 완성도에 집중한다.

| 제외 기능 | 배치 |
|---|---|
| 채팅 | Phase 2 |

## 4. 사용자 역할 및 권한

### 4-1. 역할 구조

전역 권한과 동아리별 권한을 분리한다.

| 구분 | 역할 | 설명 |
|---|---|---|
| 전역 권한 | MEMBER | 일반 학생. 가입 후 기본 권한 |
| 전역 권한 | OPERATOR_PENDING | 운영자 가입 신청 후 승인 대기 |
| 전역 권한 | OPERATOR | 총동아리회 운영자 |
| 전역 권한 | SUPER_ADMIN_PENDING | 학교 관리자 가입 신청 후 승인 대기 |
| 전역 권한 | SUPER_ADMIN | 학교 관리자 |
| 동아리별 권한 | CLUB_LEADER | 특정 동아리의 회장 |
| 동아리별 권한 | CLUB_MEMBER | 특정 동아리의 일반 부원 |

### 4-2. 권한 원칙

- 모든 사용자는 앱에서 직접 가입할 수 있다.
- 기본 가입자는 `MEMBER` 상태로 시작한다.
- 운영자와 학교 관리자는 가입 유형을 선택할 수 있지만, 승인 전에는 제한 상태로 둔다.
- 운영자 승인은 기존 운영자 또는 학교 관리자가 처리한다.
- 학교 관리자 승인은 기존 학교 관리자 또는 초기 시드 관리자 계정이 처리한다.
- 회장 권한은 운영자가 특정 동아리에 대해 부여한다.
- 한 사용자는 여러 동아리에 속할 수 있고, A 동아리에서는 회장, B 동아리에서는 일반 부원일 수 있다.

### 4-3. 권한별 접근

| 기능 | MEMBER | CLUB_LEADER | OPERATOR | SUPER_ADMIN |
|---|---:|---:|---:|---:|
| 동아리 탐색 | O | O | O | O |
| 지원서 제출 | O | O | O | O |
| 내 지원 내역 확인 | O | O | O | O |
| 홍보카드 관리 | X | 자기 동아리만 | X | 보기 가능 |
| 지원서 검토 | X | 자기 동아리만 | 보기 가능 | 보기 가능 |
| 동아리 생성 | X | X | O | O |
| 회장 권한 부여 | X | X | O | O |
| 운영자 승인 | X | X | 제한적 | O |
| 관리자 접근 로그 확인 | X | X | 제한적 | O |

## 5. 회원가입 및 인증

### 5-1. 가입 조건

- `@sunmoon.ac.kr` 이메일만 가입 가능하다.
- 이메일 인증 완료 전에는 서비스 주요 기능에 접근할 수 없다.
- 가입 시 언어를 선택한다: 한국어 또는 영어.
- 가입 시 프로필을 입력한다: 이름, 학번, 학과, 국적.
- 가입 유형은 일반 학생, 운영자, 학교 관리자 중 선택 가능하다.

### 5-2. 가입 플로우

```text
앱 실행
  -> 언어 선택
  -> 회원가입
  -> @sunmoon.ac.kr 이메일 입력
  -> 비밀번호 입력
  -> 이메일 인증
  -> 프로필 입력: 이름, 학번, 학과, 국적
  -> 가입 유형 선택
  -> 일반 학생은 홈 이동
  -> 운영자/학교 관리자는 승인 대기 화면 이동
```

### 5-3. 로그인 방식

- MVP는 Firebase Authentication 이메일/비밀번호 방식을 사용한다.
- 이메일 인증이 완료되지 않은 계정은 로그인 후 인증 안내 화면으로 이동한다.
- 비밀번호 재설정은 Firebase Auth의 이메일 재설정 기능을 사용한다.
- 관리자 계정은 2단계 인증을 추후 적용 가능 항목으로 명시한다.

## 6. 주요 화면 정의

### 6-1. 일반 학생 하단 탭

| 탭 | 목적 | 주요 요소 |
|---|---|---|
| 홈 | 내 활동 요약 | 내 동아리, 최근 공지, 지원 상태 요약, 권한별 대시보드 진입 |
| 탐색 | 동아리 찾기 | 검색, 카테고리 필터, 모집 중 필터, 언어 필터, 홍보카드 목록 |
| 지원내역 | 내가 제출한 지원서 확인 | 동아리명, 제출일, 상태, 상세 보기 |
| 알림 | 앱 내부 알림함 | 지원 상태 변경, 공지, 관리자 알림, 읽음 처리 |
| 마이페이지 | 계정/언어/권한 확인 | 프로필, 언어 설정, 가입 유형/승인 상태, 로그아웃 |

### 6-2. 동아리 상세

| 화면 | 주요 요소 |
|---|---|
| 동아리 상세/홍보 페이지 | 대표 이미지, 동아리명, 카테고리, 모집 상태, 안전/검토 배지, 소개, 활동 사진, 모집 요강, 번역 보기, 지원하기 버튼 |
| 지원서 작성 | 필수 항목 이름/학번 자동 입력 또는 확인, 회장 설정 질문, 필수 여부 검증, 제출 버튼 |
| 지원 완료 | 제출 완료 안내, 지원 상태, 지원내역 이동 버튼 |

### 6-3. 회장 대시보드

회장 기능은 홈 또는 마이페이지의 권한별 진입 버튼을 통해 접근한다.

| 화면 | 주요 요소 |
|---|---|
| 회장 홈 | 내 동아리 요약, 지원자 수, 모집 상태, 빠른 작업 |
| 홍보카드 관리 | 대표 이미지, 한줄소개, 태그, 모집 상태, 소개글, 저장 |
| 지원서 양식 설정 | 질문 추가/삭제, 질문 타입, 필수 여부, 순서 변경 |
| 지원자 관리 | 지원자 목록, 상태 필터, 지원서 상세, 합격/불합격 처리 |

### 6-4. 관리자 대시보드

운영자와 학교 관리자는 MVP에서 앱 내부 대시보드를 사용한다. 웹 대시보드는 Phase 3 이후로 둔다.

| 사용자 | 화면 | 주요 요소 |
|---|---|---|
| 운영자 | 동아리 관리 | 동아리 목록, 생성, 비활성화 |
| 운영자 | 회장 권한 부여 | 학생 검색, 동아리 선택, 회장 지정 |
| 운영자 | 승인 대기 검토 | 운영자/관리자 신청 상태 확인, 권한 범위 내 승인 |
| 학교 관리자 | 운영자 관리 | 운영자 승인/정지/복구 |
| 학교 관리자 | 관리자 로그 | 관리자 접근/권한 변경 로그 확인 |

### 6-5. Stitch 디자인 참고 반영

유지 후보:

- 네이비/골드 중심 컬러 시스템
- 카드형 리스트
- 칩/배지 기반 필터
- 하단 탭 구조
- 알림 카드 UI
- 동아리 홍보카드 이미지 중심 레이아웃

삭제 또는 Phase 2 참고:

- 캠퍼스 피드 중심 홈
- 만들기 FAB
- 채팅 중심 탭
- SNS형 좋아요/댓글/공유 액션
- 회비 장부 화면

## 8. 번역 정책

### 8-1. UI 번역

- UI 텍스트는 Google Translate API로 실시간 번역하지 않는다.
- 앱 UI는 i18n 정적 파일로 관리한다.
- MVP 필수 언어는 한국어와 영어다.
- 모든 버튼, 탭, 오류 메시지, 상태 텍스트는 i18n 키로 관리한다.

예시:

```text
ko.json
en.json
```

### 8-2. 콘텐츠 번역

Google Translate API는 다음 사용자 생성 콘텐츠에만 사용한다.

- 동아리 소개
- 동아리 모집 요강
- 동아리 공지

번역 결과는 Firestore에 캐시한다. 동일 콘텐츠와 동일 언어에 대해서는 재번역 호출을 최소화한다.

### 8-3. 번역 실패 처리

- 번역 API 실패 시 원문을 표시한다.
- 사용자에게 "번역을 불러오지 못해 원문으로 표시됩니다." 수준의 짧은 안내를 보여준다.
- 지원서 답변은 자동 번역 대상에서 제외한다.

## 14. 테스트 및 검수 기준

### 14-1. 학생 시나리오

```text
학교 이메일 가입
-> 이메일 인증
-> 로그인
-> 언어 선택
-> 동아리 탐색
-> 동아리 상세 보기
-> 지원서 제출
-> 지원내역에서 상태 확인
```

완료 기준:

- `@sunmoon.ac.kr` 외 이메일 가입이 차단된다.
- 필수 프로필을 모두 입력해야 가입이 완료된다.
- 모집 중인 동아리만 지원 가능하다.
- 중복 지원이 차단된다.
- 지원내역에 제출한 지원서가 표시된다.

### 14-2. 회장 시나리오

```text
회장 권한 계정 로그인
-> 회장 대시보드 진입
-> 홍보카드 작성/수정
-> 지원서 양식 설정
-> 지원자 목록 확인
-> 지원서 상세 열람
-> 합격/불합격 처리
-> 지원자 알림 생성 확인
```

완료 기준:

- 회장은 자기 동아리만 관리할 수 있다.
- 지원서 상태 변경이 저장된다.
- 상태 변경 후 지원자 알림이 생성된다.
- 양식 변경 후에도 기존 제출 지원서의 스냅샷이 유지된다.

### 14-3. 운영자 시나리오

```text
운영자 승인 계정 로그인
-> 관리자 대시보드 진입
-> 동아리 생성
-> 일반 학생 검색
-> 특정 동아리 회장 권한 부여
-> AuditLog 기록 확인
```

완료 기준:

- 승인되지 않은 운영자 계정은 관리자 기능에 접근할 수 없다.
- 운영자는 동아리를 생성할 수 있다.
- 회장 권한 부여 후 대상 학생은 회장 대시보드에 접근할 수 있다.
- 권한 변경은 AuditLog에 기록된다.

### 14-4. 권한 테스트

- 일반 학생은 회장/운영자 화면에 접근할 수 없다.
- 회장은 자기 동아리 외 지원서를 열람할 수 없다.
- 운영자는 권한 범위 밖의 관리자 작업을 수행할 수 없다.
- 정지 계정은 주요 기능 접근이 제한된다.

### 14-5. 예외 테스트

- 잘못된 이메일 도메인 입력
- 이메일 미인증 로그인
- 필수 지원서 항목 누락
- 중복 지원
- 번역 API 실패
- 네트워크 오류
- 승인 대기 운영자/관리자 계정 접근

## 16. 개발 시 주의사항

- PRD v2가 MVP 구현의 기준 문서다.
- 기존 Claude 문서의 회비, FCM, 채팅 중심 기능은 현 MVP 기준에서 제외한다.
- 기존 Expo 코드의 화면은 재사용보다 재설계 대상이다.
- Firestore 보안 규칙은 화면 권한보다 우선해야 한다.
- 모든 UI 텍스트는 i18n 키로 관리한다.
- Google Translate API 키는 클라이언트에 직접 노출하지 않는 방향을 우선 검토한다. MVP에서 불가피하게 클라이언트 호출을 사용하는 경우 키 제한을 반드시 적용한다.
- 지원서와 관리자 로그는 개인정보와 권한 이슈가 크므로 더미 데이터 시연 때도 실제 개인정보를 넣지 않는다.
