# requirements.md

> UniMeet — 기능 및 비기능 요구사항 정의서  
> 문서 버전: v1.0 | 기준: MVP (7~8주차, 2주 구현)

---

## 1. 문서 목적

이 문서는 UniMeet MVP 개발에서 반드시 충족해야 하는 기능 요구사항과 비기능 요구사항을 정의한다. 팀원 간 구현 범위 합의, 개발 우선순위 결정, 발표 시연 기준 확인을 위해 작성되었다. 모든 요구사항은 실제 구현 가능성을 기준으로 작성되었으며, MVP 범위를 벗어나는 항목은 명시적으로 제외 처리한다.

> **우선순위 기준**  
> `Must` — MVP에 반드시 포함. 없으면 발표 시연 불가  
> `Should` — 가능하면 구현. 완성도에 영향  
> `Could` — 시간 여유 시 추가. 없어도 MVP 검증 가능

| REQ-CM-05 | 운영진은 미납 부원에게 알림을 발송할 수 있어야 한다 | Must |

> "동아리장으로서, 지원자 목록을 앱에서 확인하고 합격 처리까지 한 곳에서 하고 싶다. 지금은 구글폼·카카오톡·엑셀을 따로 써야 해서 불편하기 때문이다."

## 6. 비기능 요구사항

### 6-1. 성능

| ID | 요구사항 | 기준 |

## 9. 성공 기준

MVP 성공 여부는 아래 기준으로 판단한다.

### 기술적 성공 기준

| 기준 | 측정 방법 |
|---|---|
| ✅ 유학생 계정으로 로그인 → 모국어 동아리 탐색 → 지원서 제출 시나리오가 오류 없이 완료된다 | 발표 시연 라이브 테스트 |
| ✅ 운영진 계정에서 지원자를 합격 처리하면 해당 지원자에게 푸시 알림이 발송된다 | 발표 시연 라이브 테스트 |
| ✅ 동아리 공고가 한국어 원문에서 다른 언어로 자동 번역되어 표시된다 | 발표 시연 라이브 테스트 |
| ✅ 회비 내역 등록 및 미납 알림 발송이 동작한다 | 발표 시연 라이브 테스트 |

### 서비스 관점 성공 기준

| 기준 | 내용 |
|---|---|
| ✅ 핵심 가설 검증 | "유학생이 한국어 없이도 동아리를 탐색하고 지원할 수 있는가"에 대한 시나리오 완성 |
| ✅ 사용자 시나리오 3개 완성 | 유학생 탐색·지원 / 내국인 탐색 / 운영진 관리 시나리오 모두 시연 가능 |
| ✅ Must 요구사항 100% 구현 | 위에 정의된 Must 항목 전체 구현 완료 |
